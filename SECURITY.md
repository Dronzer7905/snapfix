# Security Policy

## Supported Versions

The following versions of **Snapfix** are currently being supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2.0 | :x:                |

## Reporting a Vulnerability

We take security seriously and appreciate your efforts to responsibly disclose vulnerabilities.

If you find a security bug in Snapfix, please **do not open a public issue**. Instead, follow these steps:

1.  **Email**: Send a detailed report to **[security@dronzer.me]**.
2.  **Wait**: Please allow us up to 48 hours to acknowledge your report.
3.  **Triage**: We will investigate the issue and coordinate a fix.
4.  **Disclosure**: Once a fix is released, we will publicly disclose the vulnerability if appropriate.

### What to Include
- A clear description of the vulnerability.
- Steps to reproduce (including a proof-of-concept if possible).
- The potential impact of the issue.
- Your preferred name or handle for credit in our security advisory (optional).

### Scope
This policy applies to the **Snapfix Engine** (backend) and the **Snapfix VS Code Extension**.
